## 📌 Sobre o Projeto
Após a fase de análise exploratória, o desafio na **Telecom X** evoluiu para a construção de soluções de **Inteligência Artificial**. O objetivo é prever, através de algoritmos de classificação, quais clientes possuem maior probabilidade de cancelar seus serviços (Churn).

Este repositório documenta todo o pipeline de Machine Learning: desde a preparação avançada dos dados até a comparação de modelos preditivos.

---

## ⚙️ Pipeline de Machine Learning

### 1. Pré-processamento e Engenharia
* **Limpeza e Seleção:** Remoção de colunas irrelevantes para otimizar o aprendizado.
* **Encoding:** Transformação de variáveis categóricas em numéricas via *One-Hot Encoding*.
* **Tratamento de Dados:** Avaliação de balanceamento de classes e aplicação de técnicas de normalização/padronização para modelos baseados em distância.



### 2. Modelagem Preditiva
* **Algoritmos Selecionados:** Comparação entre modelos de naturezas distintas (ex: Regressão Logística vs. Random Forest).
* **Validação:** Divisão de dados (Treino/Teste) e análise de performance (Matriz de Confusão, Acurácia, Recall e F1-Score).



---

## 🧠 Insights e Interpretabilidade
* **Análise de Correlação:** Identificação de variáveis que possuem maior impacto direto na decisão de evasão.
* **Feature Importance:** Investigação de quais atributos (ex: tempo de contrato, total gasto) foram os mais determinantes para as previsões dos modelos.
